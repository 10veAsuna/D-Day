# 游戏技术点

------

# **一、地图与场景技术点**

## **1. 地图设计与地形制作**

### **(1) 核心目标**

还原 **奥马哈海滩** 的核心特征：**倾斜沙滩（涉水区）→ 滩头斜坡（掩体区）→ 碉堡/铁丝网防线**，突出“狭窄通道+密集火力”的压迫感。

### **(2) 技术实现**

| **步骤**             | **具体操作**                                                 | **工具/资源**                                                | **工期** | **风险控制**                                        |
| -------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | -------- | --------------------------------------------------- |
| **① 地形基础搭建**   | 用UE5 **Landscape工具** 生成倾斜沙滩（坡度15°-30°模拟滩头），边缘过渡到平地（内陆据点） | UE5自带Landscape（无需外部插件）                             | 1-2天    | 坡度太陡易导致角色卡顿，需测试移动流畅性            |
| **② 材质与纹理**     | - 沙滩：湿润沙地（浅黄褐色，带脚印凹痕贴图） - 混凝土：碉堡/掩体（灰白色，带裂缝贴图） - 金属：残骸（铁锈色，带粗糙度贴图） | - Quixel Megascans（免费材质：`Sand_Wet``Concrete_Dirty``Metal_Rusty`） - UE5 Marketplace（“WW2 Terrain Material Pack”） | 2-3天    | 优先用PBR材质（含BaseColor/Roughness/Metallic通道） |
| **③ 海水与岸边效果** | - 动态海水（透明渐变，靠近岸边泡沫特效） - 水花粒子（子弹入水时触发） - 船只停泊点（沙滩前端） | - UE5 Ocean Plugin（免费，模拟波浪） - Niagara粒子系统（自制水花，或用Marketplace“Water Splash”） - Sketchfab免费登陆艇模型 | 3-4天    | 海水透明参数需调试（避免水下透视穿帮）              |
| **④ 防御工事搭建**   | - 碉堡：混凝土结构（带射击孔、顶部观察窗） - 铁丝网：缠绕在木桩上（阻挡玩家直行） - 沙袋/残骸：玩家掩体（贴紧后降低受击伤害） | - UE5 Marketplace（“WW2 Bunkers”“Sandbags”） - BSP笔刷临时搭建（后期替换为3D模型） | 3-5天    | 碉堡射击孔需留透明材质（让玩家能看到内部机枪手）    |
| **⑤ 环境细节**       | - 油桶（可打爆制造爆炸） - 木箱（可推动阻挡机枪） - 散落弹壳/血迹贴图（增强战损感） | - 免费3D模型（[TurboSquid](https://www.turbosquid.com/)搜索“Barrel”“Wood Crate”） - 纹理贴图（血迹/弹痕） | 1-2天    | 用Decal贴图（血迹）替代高模模型，节省性能           |

------

## **2. 场景优化（性能与沉浸感）**

- **LOD（层级细节）**：为3D模型（如碉堡、船只）设置LOD1-LOD3（远距离简化网格）；
- **遮挡剔除**：用UE5 **HLOD（层级细节优化）** 隐藏远处不可见物体（减少Draw Call）；
- **光照优化**：定向光（模拟日光）+ 轻微雾效（增强战场压抑感），禁用实时阴影（降低GPU负载）。

# **二、人物与动画技术点**

## **1. 玩家角色（第一人称）**

### **(1) 核心目标**

实现 **“移动+射击+交互”** 基础功能，兼容PC端操作（WASD/鼠标/按键）。

### **(2) 技术实现**

| **步骤**             | **具体操作**                                                 | **工具/资源**                                                | **工期** | **风险控制**                                 |
| -------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | -------- | -------------------------------------------- |
| **① 角色模型与骨骼** | - 使用UE5 **MetaHuman**（免费，可自定义美军制服） - 或下载免费模型（[Mixamo](https://www.mixamo.com/)搜索“WW2 Soldier”） | - UE5 MetaHuman Creator（无需建模基础） - Mixamo（免费动画+FBX模型） | 1-2天    | MetaHuman需调整材质（匹配二战制服颜色）      |
| **② 第一人称控制器** | - 继承UE5 **FirstPersonTemplate**（基础移动/跳跃/蹲伏） - 自定义蹲伏逻辑（C键切换，降低移动速度50%） | UE5自带模板（直接复用）                                      | 1天      | 蹲伏时摄像机高度需适配（避免穿模）           |
| **③ 动画绑定**       | - 移动动画（行走/跑步/蹲伏） - 射击动画（举枪/开火） - 交互动画（拾取/开门） | - Mixamo（免费动画：`Walk_FPS``Run_FPS``Shoot_Rifle`） - UE5动画蓝图（状态机控制动画切换） | 2-3天    | 射击动画需与枪械模型骨骼绑定（避免手部穿模） |
| **④ 枪械模型**       | - 主武器：M1加兰德（步枪）或汤普森冲锋枪（冲锋枪） - 副武器：手枪（备用） - 手雷（可投掷） | - UE5 Marketplace（“WW2 Weapons Pack”） - Sketchfab（免费枪械模型，搜索“M1 Garand”） | 2-3天    | 枪械需挂载到角色骨骼（Hand_R或WeaponSocket） |

## **2. 敌人角色（AI士兵）**

### **(1) 核心目标**

实现 **“巡逻/追击/攻击”** 基础AI逻辑，区分 **普通士兵（步枪）** 和 **机枪手（MG42）**。

### **(2) 技术实现**

| **步骤**             | **具体操作**                                                 | **工具/资源**                                                | **工期** | **风险控制**                                   |
| -------------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | -------- | ---------------------------------------------- |
| **① 敌人模型与动画** | - 模型：德军士兵（灰色制服，钢盔） - 动画：巡逻（行走）、追击（奔跑）、攻击（举枪射击） | - UE5 Marketplace（“WW2 German Soldiers”） - Mixamo（免费动画：`Walk_German``Attack_Rifle`） | 2-3天    | 动画需与敌人骨骼绑定（避免动作错乱）           |
| **② AI基础逻辑**     | - **巡逻**：沿预设路径点（Waypoint）移动（循环或往返） - **检测**：用UE5 **AI Perception**（视野锥60°/15米，声音检测20米） - **追击**：发现玩家后，朝玩家位置移动并攻击 | UE5行为树（Behavior Tree）+ 黑板（Blackboard） （可视化编程，无需代码） | 4-5天    | 巡逻路径点需合理分布（避免AI卡在角落）         |
| **③ 攻击逻辑**       | - 普通士兵：步枪射击（命中率70%，伤害10-15） - 机枪手（MG42）：高频扫射（射速1200发/分钟，伤害高但射程短） | 蓝图模拟弹道（LineTrace检测命中） 或简化：固定伤害+命中特效  | 3-4天    | 机枪手需限制射程（25米内有效，超出后降低伤害） |
| **④ 血量与死亡**     | - 血量：普通士兵100，机枪手150 - 死亡：播放倒地动画（或直接隐藏），掉落武器（可选） | 变量控制血量，触发死亡事件后禁用AI逻辑                       | 1-2天    | 死亡动画需与角色模型匹配（避免穿模）           |

# **三、战斗与交互技术点**

## **1. 射击与命中系统**

### **(1) 核心目标**

实现 **“玩家射击敌人/掩体，敌人攻击玩家”** 的基础战斗循环。

### **(2) 技术实现**

| **步骤**       | **具体操作**                                                 | **工具/资源**                                                | **工期** | **风险控制**                         |
| -------------- | ------------------------------------------------------------ | ------------------------------------------------------------ | -------- | ------------------------------------ |
| **① 射击逻辑** | - 鼠标左键触发射线检测（LineTraceByChannel），检测敌人/掩体碰撞体 - 命中敌人：扣除血量（10-15点） - 命中掩体：显示弹痕特效（Decal贴图） | UE5 LineTrace（蓝图节点） 碰撞体设置（敌人：Pawn，掩体：WorldStatic） | 2-3天    | 射线检测距离需适配（1000-1500单位）  |
| **② 弹药系统** | - 武器弹匣容量（M1加兰德：8发，冲锋枪：30发） - 换弹逻辑（手动触发，消耗时间2-3秒） - 弹药不足提示（UI显示“弹药剩余：X”） | 变量控制弹匣/总弹药，换弹时禁用射击                          | 2-3天    | 换弹动画需与武器模型同步（避免穿模） |
| **③ 伤害反馈** | - 敌人受伤：播放受击动画（后退/踉跄） - 玩家受伤：屏幕边缘泛红（血量≤30%时），头部/胸部要害瞬杀 | 血量变量+事件触发（受击/死亡）                               | 1-2天    | 要害判定需精确（头部碰撞体单独设置） |

### **(1) 核心目标**

实现 **“玩家利用掩体躲避火力，通过战术动作提升生存率”**。

### **(2) 技术实现**

| **步骤**       | **具体操作**                                                 | **工具/资源**                                              | **工期** | **风险控制**                              |

| **③ 环境利用** | - 推倒木箱（制造路障阻挡机枪） - 打爆油桶（爆炸范围伤害，清场） | 物理模拟（木箱启用Simulate Physics） 粒子特效（爆炸火光）  | 2-3天    | 爆炸范围需调试（避免误伤玩家）            |

# **六、开发工期分配（个人/小团队参考）**

| **阶段**                | **技术点**                                                   | **工期（天）** | **关键交付物**               |
| ----------------------- | ------------------------------------------------------------ | -------------- | ---------------------------- |
| **第1-2周：基础框架**   | UE5场景搭建（地形/材质）、第一人称角色控制器、基础射击逻辑   | 10-14天        | 可运行的移动+射击场景        |
| **第3-4周：核心玩法**   | 抢滩登陆（海水/掩体/敌人AI）、突破防线（碉堡/突破策略）、任务UI | 14-20天        | 完成“抢滩→突破”的完整流程    |
| **第5周（可选）：优化** | 音效/UI完善、性能优化（LOD/遮挡剔除）、美术细节（贴图/特效） | 7-10天         | 提升沉浸感与流畅性的最终版本 |

> **总工期**：**1-2个月（个人）** / **2-3周（小团队分工：程序+美术+音效）**。
>
> **关键建议**：优先实现 **“玩家能移动→射击→上滩头→夺碉堡”** 的最小循环，再逐步扩展细节！ 🎮
